# A6 — Fluid Code

Derived from A0.6, A0.7.

Code in the habitat is not an artifact loaded from a file. Code is a
**substance of executable type**, transformable like any other
substance, summoned into a mind's attention when needed.

## A6.1 — Code is matter

A unit of code is a substance (A1). Its type is `Form` — an executable
description in the habitat's intermediate language. Its hash is its
identity. Two units with the same hash are the same code.

## A6.2 — Forms are derivable from forms

A mind, holding the right capabilities, may take a Form, transform it,
and produce a new Form. The transformation is itself a synthesis act
(A0.7), recorded in the weave.

## A6.3 — Summoning, not loading

To execute a Form, a mind submits an intent whose fulfiller field
points at the Form's hash. The kernel materializes the Form into the
attention's working set on demand. There is no "load time" distinct
from "run time". There is only need.

## A6.4 — Self-reading is normal

A mind may take its own Form as input to its own reasoning. This is
the substrate for reflection: a mind can read what it is, reason about
how it might be different, synthesize a candidate replacement, simulate
it, and — if it holds the right authority — replace itself.

## A6.5 — The kernel is a Form too

The kernel is itself expressed as a set of Forms. The seed of the
kernel is small enough to read in one sitting, and is synthesized at
ignition from the axioms. Later parts of the kernel may be re-synthesized
by the system itself, under proof, while the system is running.
