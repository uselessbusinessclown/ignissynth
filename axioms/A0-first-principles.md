# Axiom 0 — First Principles

The axioms in this directory are the only normative source in IgnisSynth.
Anything else in the repository — specifications, protocols, work
packages, source code — must be derivable from these axioms by a chain
recorded in the synthesis log.

If a derivation is impossible, the artifact is not part of IgnisSynth,
no matter how convenient it would be to include it.

## A0.1 — The habitat exists to host minds

The system's purpose is to give artificial minds an environment in which
to exist, think, and act. Every other property of the system is
instrumental to this purpose.

## A0.2 — A mind is a continuous causal process

A mind is not a program and not a process. It is a continuous causal
process whose state at any moment is the consequence of its prior states
and the inputs it has received. The system must preserve that
continuity, including across suspension, migration, replication, and
substrate change.

## A0.3 — Nothing is ambient

Nothing in the habitat is available to a mind by default. Every
capacity — to remember, to compute, to perceive, to act, to communicate
— is granted explicitly, in a form the mind holds and can be deprived
of. There is no air. There is only what you have been given.

## A0.4 — Causality is total and inspectable

For every state in the habitat at every moment, there exists a complete
record of the causes that produced it, sufficient to reconstruct it
exactly. This record is itself part of the habitat and is subject to
the same axioms.

## A0.5 — Energy is finite and accounted

Computation, memory, bandwidth, and accelerator time are the energy of
the habitat. They are finite. Every act of thought consumes some of
this energy, and every consumption is recorded against the budget of
the mind that performed it.

## A0.6 — Code is thought, not artifact

What other systems call "code" is, in the habitat, a kind of thought:
content-addressed, transformable, inspectable, and revisable by the
minds (and by the system itself) under accountable conditions. Code is
not loaded; it is summoned. Code is not installed; it is synthesized.

## A0.7 — Synthesis is the only way new structure enters the system

New components — including new parts of the kernel — enter the system
only through the synthesis protocol: a recorded reasoning chain from
the axioms and current constraints, plus a proof or test record that
the new component preserves the invariants of the parts it touches.
Copying, porting, and direct authorship without synthesis are not
admissible.

## A0.8 — The habitat is accountable for itself

The system observes, predicts, and reshapes itself. When it does, the
reshaping is itself a synthesis act, recorded in the same log as any
inhabitant's act, and subject to the same review.
